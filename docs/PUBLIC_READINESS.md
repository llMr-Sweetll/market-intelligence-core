# Public Readiness Checklist

The repository should become public only after this checklist passes on `main`.

## Required

- [ ] License is source-available and noncommercial.
- [ ] README states the project is not open-source licensed.
- [ ] Contribution guide exists.
- [ ] Security policy exists.
- [ ] Support policy exists.
- [ ] Pull request template exists.
- [ ] Issue templates exist.
- [ ] CODEOWNERS exists.
- [ ] Release plan exists.
- [ ] No credentials, tokens, account IDs, or secrets are present.
- [ ] No direct tool-attribution references are present in tracked files.
- [ ] No stale private-only wording remains.
- [ ] CI passes.
- [ ] API smoke test passes.
- [ ] Migration verification passes.

## Public Visibility Notes

When the repository is public:

- code is visible to anyone
- forks can be created
- Actions logs are visible
- public issues and pull requests are visible

Do not switch visibility until the license and public-facing documentation are
merged.
